Sort items with Merge Sort - Divide & Conquer!

Merge Sort is an algorithm that uses a divide and conquer approach. A list (array, ArrayList instance) of items is split into 2 parts, which are then spit into 2 parts, and so on until the entire list of items have been split into singletons. The singletons are then merged together to form sub-lists that are sorted. These sorted lists are merged successively to form a final, sorted list.


Sample Code for Merge Sort, Assuming Array Of Ints - nums

public static void mergeSort(int[] nums, int n) {
    if (n < 2) 
        return;
    int mid = n / 2;
    int[] left = new int[mid];
    int[] right = new int[n - mid];
    
    for (int i = 0; i < mid; i++) 
        left[i] = nums[i];
        
    for (int i = mid; i < n; i++) 
        right[i - mid] = nums[i];
    // recursively sort the left and right
    mergeSort(left, mid);
    mergeSort(right, n - mid);
    // Merge the sub-lists
    merge(nums, left, right, mid, n - mid);
}
    
    public static void merge(
    int[] nums, int[] l, int[] r, int left, int right) {

        int i = 0, j = 0, k = 0;
        while (i < left && j < right) {
            if (l[i] <= r[j]) 
                nums[k++] = l[i++];
            else 
                nums[k++] = r[j++];
            }
        while (i < left) 
            nums[k++] = l[i++];
        while (j < right) 
            nums[k++] = r[j++];
    }  
}

How Efficient Is Merge Sort?

We say of the order of, designated by O(...), to describe the worst case in terms of comparisons. This is termed big O representation. The best case is described using big Omega notation, Ω(...).

Merge sort is a divide and conquer algorithm. There are 3 steps,

  • The divide step computes the midpoint of each of the sub-arrays. Each of this step just takes O(1) time.
  • The conquer step recursively sorts two subarrays of n/2 items
  • The merge step merges n items
The order of complexity is O(nlog2n) for the merge sort algorithm.
